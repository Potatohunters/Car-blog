About content

Our theme

The theme of our blog is about cars, with a focus on hypercars. Not much is known about hypercars, perhaps because of their high prices. But there's more to hypercars than just their cool looks and high prices. Each hypercar has its own unique design, containing the ideas of the designer and its own brand culture. Special tailplane design, engine layout, engine performance, and aerodynamics are all worth discussing. We believe that hypercar is a combination of physics and design aesthetics, so we chose the top sports car as the topic of our blog

Password policy

Our password policy consists of three main parts. The registration password must contain at least one uppercase letter, at least one number, and at least eight characters. The password must be the same as the confirmation password. To accomplish these goals, we first add the "li" tag to the HTML file, enter the required password policy into the registration box, and give it the same id. And inside it calls the "checkword" and "recheckword" methods with "onkeyup" and reads the data with "(this.value)". After that, we use regular expressions in the js file to determine whether the string has uppercase letters, whether it contains numbers, and whether it is more than eight digits. if so, add "valid" to the original" class ". Finally, set the text style of "valid" to be transparent in the css file so that the user knows that the password is valid. The password policy for confirming the password is determined by comparing the data read twice” (this.value)”. If the data read twice is the same, the Confirm password label will be hidden.

Registration method

For new user registration, we decided to animate the login form into the registration form. In this way, users can feel the interactivity of the web page when using, and can choose freely in the registration and login, rather than the stiff jump to another separate interface. Immersive experiences are important to us. The dynamic changes can also make it more interesting.

Our Embellishments

Responsiveness

We achieved that the site dynamically adapts for different browser sizes. Through grid layout auto and media function, the layout of a blog post can change with the size of the browser. When shrinking the page, if the browser is not large enough to display the content at the minimum size, the layout will be changed again, for example, from a row of four columns to a row of three columns. But on the Home page and About page, we use a one-row, two-column layout. The navigation bar also uses a responsive design.

Read More

Sticky navigation

We use "position:fix" to keep the navigation bar anchored at the top of the page and let its top is 0 and left is 0. So when we scroll down, it position will not change and always displays at the top of our page. To avoid other content that might overwrite the navigation content, we also set it to the highest z-index. In addition, in order to achieve responsive web pages, we also set media in the navigation bar to change the way it is displayed on pages that are too small. In the beautify aspect, transform: translateX is also used to make it can achieve "slide into" animation effect when clicked.

Scroll-to-top button

When you click on it, the page is automatically pulled to the top. In the js file, we set a judgment condition, when the page position is pulled down, that is, the page position is greater than 100, the scroll up button will appear, but when the page is not pulled down, it will not appear. Such a design is more in line with people's usage habits. When the user clicks that button, we're going to use addEventListener and implement its content, and the page goes back to top=0

Read More

Dynamic display of scroll-to-top button

To increase interactivity, we added additional appearance styles for the scroll up button. Calculate the height of how much has the user pulled down in the js file, then use conic-gradient to convert it into a pie chart rotated around the center point, and then set it to the background color of the button. In this way, users can more intuitively know how far they have pulled down.

Light and dark themes

For changing the theme color of the blog, the method we use is to store all the colors that need to be changed as a class in CSS. Create a light mode class and a dark mode class. The background color and color of the element are then called in CSS using var (). JavaScript only needs to change the light and dark classes to make the color switch

Full screen display mode on archive page

In the archive page, checkbox type and label tags are added for each blog card. When the card is clicked, the full screen expansion of the card and the display of the content is achieved by the transform function in css. In addition, the transformation of each card is animated by the transition function in CSS.
